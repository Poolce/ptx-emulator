\section{Заключение}

\subsection{Итоги работы}

В рамках данной выпускной квалификационной работы были достигнуты
все семь задач, сформулированных в разделе~1.3.

\textbf{1. Механизм перехвата CUDA Runtime.}
Реализована динамическая библиотека \texttt{libemuruntime.so},
перехватывающая 8~функций CUDA Runtime API через механизм
\texttt{LD\_PRELOAD}. Разработан CLI-лончер \texttt{cuemu},
позволяющий запускать произвольный CUDA-бинарный файл под
управлением эмулятора единственной командой без модификации
исходного кода. Прозрачность перехвата верифицирована тестом
«один бинарный файл~--- два режима».

\textbf{2. Синтаксический анализатор PTX.}
Реализован однопроходный инкрементный PTX-парсер, поддерживающий
подмножество PTX ISA~7.x: 66~мнемоник 7~классов. Решена проблема
экспоненциального бэктрекинга \texttt{std::regex} при разборе
PTX-функций размером более 30~КБ; разработан линейный сканер
с явным счётчиком вложенности. Корректность верифицирована
на 7~интеграционных ядрах.

\textbf{3. Таблица диспетчеризации и исполнители.}
Построена таблица диспетчеризации с O(1)-поиском по хешу мнемоники.
Реализованы исполнители для всех 66~поддерживаемых инструкций
с формальной операционной семантикой в нотации SOS. Разработана
система кодогенерации на основе \texttt{instructions.json} и
\texttt{autogen.py}, исключающая рассинхронизацию между парсером
и исполнителями.

\textbf{4. Алгоритм PDOM-реконвергенции.}
Реализован алгоритм PDOM\_Diverge со стеком фреймов реконвергенции
типа \texttt{StackFrame}. Максимальная глубина стека при $k$~уровнях
дивергенции: $|S| \leq 2k$. Для всех 7~интеграционных тестов
алгоритм корректно обрабатывает вложенные дивергенции, дивергентные
циклы и ранний выход потоков.

\textbf{5. Система метрик профилировщика.}
Реализованы четыре метрики: \texttt{branch\_efficiency}
($\eta_{\mathrm{branch}} = \popcount(M)/32$),
\texttt{bank\_conflicts} (формула~\eqref{eq:bank_conflicts}),
\texttt{global\_mem\_transactions} ($T_{\mathrm{actual}}$),
\texttt{global\_coalescing} ($\kappa$~--- формула~\eqref{eq:coalescing}).
Кодогенерируемая архитектура позволяет добавить новую метрику
без изменения C++-кода.

\textbf{6. HTML-отчёт с аннотацией.}
Разработана трёхкомпонентная система Python-инструментов:
\texttt{parser.py}, \texttt{aggregator.py}, \texttt{renderer.py}.
Самодостаточный HTML-отчёт содержит 5~секций и аннотирует строки
исходного \texttt{.cu}-файла метриками через \texttt{.loc}-директивы.

\textbf{7. Верификация.}
Все 7~интеграционных CUDA-ядер пройдены со статусом PASS;
максимальное отклонение от CPU-эталона не превышает $10^{-3}$.
Корректность всех 4~метрик профилировщика подтверждена
аналитически.

\subsection{Ограничения}

\begin{itemize}
  \item \textbf{Неподдерживаемые инструкции.}
        Матричные инструкции \texttt{mma.m16n16k16} и
        \texttt{ldmatrix} (тензорные ядра Tensor Core),
        атомарные операции с семантикой acquire/release
        (\texttt{atom.acq\_rel}).
  \item \textbf{Упрощённая модель банков.}
        Не реализован broadcast-режим Ampere (обращение нескольких
        потоков к одному адресу не вызывает конфликта); текущая
        реализация считает это конфликтом 32 степени.
  \item \textbf{Отсутствие модели задержек.}
        Эмулятор функционально корректен, но не воспроизводит
        латентности конвейера SM.
  \item \textbf{Ограничение глубины стека вызовов.}
        Рекурсивные ядра устройства не поддерживаются.
\end{itemize}

\subsection{Направления развития}

\begin{itemize}
  \item \textbf{Поддержка Tensor Core.} Реализация
        \texttt{mma.m16n16k16.f32.f16.f16.f32}~--- ключевой
        инструкции для современных задач глубокого обучения.
  \item \textbf{JIT-ускорение.} Трансляция PTX $\to$ LLVM~IR
        с последующей JIT-компиляцией через LLVM MCJIT позволит
        снизить overhead на $1$--$2$~порядка.
  \item \textbf{Расширение метрик.} Occupancy (теоретическая
        заполненность SM), warp~efficiency (доля варпов без
        дивергенции), register~pressure.
  \item \textbf{Интеграция с GDB.} Поддержка отладки
        через RSP-протокол позволит использовать стандартные
        инструменты отладки для CUDA-ядер без GPU.
\end{itemize}
